Led by Nathan Layman

The data lakehouse and data organization

Data governance is the foundation for context engineering. This session covers the data lakehouse, medallion architecture, the Unified Data Model, and why governance determines whether AI context is trustworthy enough to use.

Module brief

Quick reference for the presenter

Learning goal

Evaluate context before using it

Participants should leave able to explain why provenance, permissions, quality, and stewardship determine whether context is safe enough to use in an AI workflow.

In-room move

Open with a dashboard analogy

Start from the governance work they already do for dashboards, then show that AI context uses the same institutional muscles under higher stakes.

Key distinction

AI both consumes data and generates new data

The system reads governed inputs but can also produce new structured outputs — extracted fields, classifications, summaries — that enter downstream workflows and need governance too.

Participant artifact

A context readiness checklist

Participants use the checklist handout to judge whether a data source is ready for AI-assisted work.

Facilitation

Speaker notes and session close

Talking points

Key messages to reinforce

  • Present governance as the condition that makes automation safe, not as bureaucracy.
  • More context can increase error surface area when institutions don't know which data is authoritative.
  • Keep returning to "trustworthy context" rather than "more context."
  • Make the consume-versus-generate distinction explicit: AI can create structured outputs that look authoritative if no one validates them.
  • Connect to what they already do: governing data for dashboards is the same muscle as governing data for AI.
Demo handoff

Vandalizer demo closes this session

Barrie Robison leads a live Vandalizer demo showing how well-organized, governed data powers real RA workflows. Then pose the bridge question: "Can you trust the output?" — handing off directly to the reproducibility module on evaluation, and accuracy.
